An existing product

My time at Developerhub was the first time I joined and existing product, working in a number of different projects: Improving the navigation on mobile, redesign of the sign up/ log in page (check it out ), history view feature, the creation of illustrations for its 404 and 403 page and a number of promotional images.

Our user base were technology companies (we call them writers) that needed to make their documentation available to their user base (readers)

Autogenerated landing page

The creation of an autogenerated landing page was one of the projects in which I worked from ideation to working software

The objetive was to provide our writing users with a modern and smart entry point where readers could understand the contents of their Developer Hub Documentation right from the start.

The challenge was for this feature to not create any extra work for the writers, having the landing page to adapt automatically to the content and auto-generate according to the structure of the documentation.

Solve one, extend to many

The problem was: with great customisation comes great difficulty to create any standards!

Our clients documentations where organised on references, docs, versions, categories, pages and subpages. Some used categories as pages and pages as subpages, others used versions as different docs… it was pretty crazy

On the colours side it was the same thing: Some users used the same (or really similar) colour for both main and links, others used white as main colours (with looks transparent) some used black for both their colours (on the right hand side we can see some examples of this)

— main takeaway, DeveloperHub

“It is possible to achieve a lot with very little. We were a team of two, working remotely, delivering this work along other personal projects. Delivering our improvements and new features every weekly update felt like a win everytime.”
